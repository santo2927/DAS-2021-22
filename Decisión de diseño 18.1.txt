# Tener una capa de Bases de Datos

* Estado: Aceptada.
* Decisores: Alejandro Fernández, Gabriel Villasevil.
* Fecha: 16/11/2021

## Es necesario una capa para gestionar los microservicios

## Motivos de la decisión.

* Motivación 1: Decision 14.1 Cambio del estilo principal MVC por estilo por capas para manejar las distintas funcionalidades de la aplicación
* Motivación 2: R3.1 Almacenar Preferencias
* Motivación 3: R3.2 Almacenar Datos de Compra
* Motivación 4: R3.4 Almacenar Localización de Microservicios
* Motivación 5: R7 Acceso a Bases de Datos
* Motivación 6: Decision 13.1 Decisión final de bases de datos


## Opciones consideradas.

* Opción 1: Tener una capa dedicada a la comunicación con las dos bases de datos SQL y NoSQL

## Decisión resultante.

Opción escogida: "Opción 1", Por el cambio de estilo de diseño en la decisión 14 podemos implementar una capa dedicada a las bases de datos para que se conecten con el negocio.

### Consecuencias positivas

* Añadir una capa enteramente dedicada a las bases de datos y la comunicación con las mimsamas.

### Consecuencias negativas

* Rehacer el estilo principal de la practica para poder implementarlo.
