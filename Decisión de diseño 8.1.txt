# Patrón mediator para el sistema de gestión de solicitudes.

* Estado: Aceptada.
* Decisores: Alejandro Fernández, Gabriel Villasevil.
* Fecha: 09/11/2021

## Es necesario un sistema que gestione las solicitudes de los usuarios al sistema.

## Motivos de la decisión.

* Motivación 1: Requisito R3.3: Gestionar Solicitudes: El sistema de la aplicación debe ser capaz de poder gestionar cualquier solicitud realizada por el usuario.

## Opciones consideradas.

* Opción 1: patrón mediator para mediar entre la base de datos y el sistema mientras se realizan las solicitudes.
* Opción 2: gestionar las solicitudes en el propio sistema.

## Decisión resultante.

Opción escogida: "Opción 1", Porque a la hora de gestionar las solicitudes tener un mediador aumenta la escalabilidad de la funcionalidad de las solicitudes.

### Consecuencias positivas

* Se podrá escalar la funcionabilidad del sistema de solicitudes de forma más eficiente.

### Consecuencias negativas

* Añadir un paso entre la base de datos y el sistema para controlar las solicitudes de manera eficiente podría ralentizar el proceso levemente.
