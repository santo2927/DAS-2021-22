R 1 Realizar Pedido: El usuario debería ser capaz de registrar uno o varios pedidos si hubiera algún producto que le interesara.
R 1.1 Realizar Compra: Una vez el usuario haya completado su pedido tendrá la opción de comprarlo si lo desea.
R 2 Realizar Devolución: El usuario debería ser capaz de devolver cualquier pedido.
R 3 Integrar Microservicios: La aplicación debe ser capaz de poder integrar microservicios para así poder mantenerse aun que se produzcan un mayor número de compras y guardar las preferencias de los usuarios.
R 3.1 Almacenar Preferencias: La aplicación debe ser capaz de almacenar los gustos de cada usuario dependiendo de las compras que hayan realizado en ella, para así mostrarle ofertas personalizadas.
R 3.2 Almacenar Datos de Compra: La aplicación debe ser capaz de almacenar los datos de cada compra (número de pedido, identificación de cliente, producto, cantidad, precio, total, forma de pago, etc.), para llevar un registro de las compras que se han realizado cada mes.
R 3.3 Gestionar Solicitudes: El sistema de la aplicación debe ser capaz de poder gestionar cualquier solicitud realizada por el usuario.
R 3.4 Almacenar Localización de Microservicios: La aplicación debe contar con una Base de Datos en MongoDB que permita guardar el lugar donde se encuentran los microservicios.
R 3.5 Comunicación Cliente-Microservicio: La aplicación contará con un componente Gateway que permitirá a los clientes conectarse con los Microservicios.
R 4 Conectar con el Sistema de pago: La aplicación debe poder conectarse con el sistema de pago previamente elegido por el usuario para asi poder realizar la compra
R 4 Sistema de Mensajería: La aplicación debe tener implementado un sistema de mensajería que le permita contactar con el usuario cuando sea necesario (notificaciones del estado del pedido).
R 5 Sistema de Seguridad:  La aplicación debe contar con un sistema de seguridad que permita al usuario realizar las compras con sus datos personales sin ningún peligro.
R 6 Acceso a Bases de Datos: La aplicación debe de contar con los componentes necesarios para poder acceder a la base de datos.
R 7 Control de Interfaz: La aplicación debe tener un componente que controle la interfaz del usuario.
R 8 Limite de intentos de compras: La aplicación debe tener la capacidad de restringir el número de intentos de compras de cada usuario a un máximo de 5 para regular el número de peticiones.
